# 已注册命令一键执行功能设计

## 目标

让用户在“已注册命令”列表中直接点击执行 DevDock 管理的命令，并在界面中看到执行结果。功能只面向已注册命令，不提供任意命令输入。

## 范围

- 在已注册命令列表每一行增加“执行”操作。
- 点击后调用 Tauri 后端执行对应命令入口。
- 后端只执行推荐命令目录中带 DevDock 管理标记的入口文件。
- 执行时不传入参数。
- 返回命令名、退出码、标准输出和错误输出。
- 前端展示执行中状态、成功或失败提示，以及最近一次执行结果。

不在本次实现中支持交互式命令、长时间后台任务、参数编辑、实时流式日志、并发队列、终止运行中进程或任意 shell 命令输入。

## 方案比较

### 方案 A：执行注册入口并捕获结果

后端根据命令名定位推荐命令目录中的入口文件，确认入口由 DevDock 生成后，用 `std::process::Command` 执行入口并捕获输出。

优点是复用现有注册机制，和用户在终端执行同一个入口一致；安全边界也清晰。缺点是长时间运行命令会等待完成。

### 方案 B：直接执行元数据里的源脚本

后端解析入口文件里的源脚本路径，绕过包装入口直接运行脚本。

优点是少一次包装跳转。缺点是行为可能和终端入口不一致，也更容易绕过未来入口层逻辑。

### 方案 C：打开系统终端执行

点击后拉起系统终端运行命令。

优点是天然支持交互式脚本。缺点是跨平台差异大、结果无法稳定回传，也会扩大权限和窗口控制范围。

## 推荐设计

采用方案 A。

后端新增 `run_registered_command(commandName)` Tauri command。它先校验命令名，再定位入口路径，然后调用现有 `read_registered_command()` 确认入口带 DevDock 标记。只有确认是 DevDock 管理的入口后才执行，不读取用户输入的任意 shell 字符串。

执行结果结构：

```ts
type CommandRunResult = {
  commandName: string;
  exitCode?: number;
  stdout: string;
  stderr: string;
};
```

前端在 `App.vue` 维护 `runningCommandName` 和 `lastRunResult`。`CommandList.vue` 接收运行中命令名和最近结果，展示“执行”按钮；正在执行的行显示 loading。列表下方展示最近一次执行结果，包含命令名、退出状态、标准输出和错误输出。

## 错误处理

- 命令名非法：返回“命令名只能包含字母、数字、点、短横线或下划线。”
- 命令不存在：返回“命令不存在。”
- 入口不是 DevDock 生成：返回“不会执行非 DevDock 生成的入口。”
- 入口执行失败：返回“命令执行失败：<系统错误>”

如果命令本身退出码非 0，后端仍返回执行结果，前端用失败提示和结果面板展示退出码与错误输出。

## 测试

- 后端测试注册一个脚本后执行入口，断言 stdout 和退出码。
- 后端测试非 0 退出码也会返回 stderr 和退出码。
- 后端测试拒绝执行非 DevDock 管理的入口。
- 前端用 TypeScript 构建验证 props、事件和状态接线。

## 安全影响

本功能会从桌面应用中运行用户已注册的本地脚本。执行范围限制在 DevDock 推荐命令目录内的 DevDock 管理入口，不支持任意命令字符串，也不传入参数。执行结果会显示在应用界面中，可能包含脚本输出的敏感信息。
